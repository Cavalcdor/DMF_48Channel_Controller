## 2. 快速上手

### 标准操作流程

```
连接串口 → 配置网格 → 设置起点/终点 → 设置障碍物 → 规划路径 → 运行路径
```

### 首次使用步骤

1. **连接硬件**：将 DMF 控制器通过 USB 连接电脑
2. **启动软件**：双击桌面快捷方式启动
3. **选择串口**：在工具栏下拉列表中选择对应的 COM 端口
4. **点击连接**：工具栏上的「连接」按钮
5. **配置网格**：确认默认 6×8 网格满足需求，否则调整行列数后点「新建」
6. **设置液滴 1**：在网格上点击设置起点（蓝色），再点击设置终点（橙色）
7. **规划路径**：点击「规划路径」预览路径
8. **运行路径**：确认路径无误后点击「执行路径」

## 3. 串口连接

### 3.1 连接串口

串口控制区域位于工具栏左侧：

```
┌──────────────────────────────────────┐
│ 端口:[COM3     ▾] [刷新] [连接]      │
└──────────────────────────────────────┘
```

1. 点击「刷新」扫描可用串口
2. 从下拉列表选择目标串口（如 COM3）
3. 点击「连接」—— 成功时状态变为「● 串口已连接」

### 3.2 断开串口

- 再次点击「连接」按钮（此时变为按下状态），即可断开

### 3.3 连接状态指示

| 状态 | 顶栏标识 | 左侧面板 |
|------|---------|---------|
| 未连接 | ● 系统就绪（绿色） | 未连接（红色） |
| 已连接 | ● 串口已连接（绿色） | 已连接 (COMx)（绿色） |
| 连接失败 | ● 连接失败（橙色） | 连接失败（橙色） |
| 运行中 | ● 液滴x 运行中（琥珀色） | — |

### 3.4 串口协议

软件支持的指令协议（发送到 Arduino/下位机）：

| 指令 | 格式示例 | 说明 |
|------|---------|------|
| 开启电极 | `ON,0` 到 `ON,47` | 开启指定编号的继电器 |
| 关闭电极 | `OFF,0` 到 `OFF,47` | 关闭指定编号的继电器 |
| 全部开启 | `ALLON` | 开启所有 48 个继电器 |
| 全部关闭 | `ALLOFF` | 关闭所有继电器 |
| 测试模式 | `TEST` | 下位机自检 |
| 列表 | `LIST` | 列出当前状态 |
| 帮助 | `HELP` | 下位机返回帮助信息 |

> **通道编号映射**：电极索引 = `行号 × 总列数 + 列号`（0-based），例如 6×8 网格中 (row=1, col=2) 对应的硬件索引为 `1×8+2=10`。

```
┌──────────────────────────────────────────┐
│  行: [ 6 ] ▲ ▼    列: [ 8 ] ▲ ▼  [新建] │
│                                          │
```

- **行/列调节**：点击 ▲▼ 按钮，范围 2×2 至 16×16
- **新建**：按当前行/列值重建网格（会清除所有现有配置）
- **清除障碍**：仅移除所有障碍物（黑色格子）
- **重置**：清除所有配置（起点/终点/障碍物/路径），回到液滴 1

### 4.2 网格状态颜色

| 状态 | 颜色 | 网格显示标记 |

|------|------|---------|

| 障碍物 | [障碍物] | 红色底白字 |

## 5. 液滴配置

### 5.1 液滴导航

```
编号: ◀ [1] ▶ [下一个]    [回到1]
```

- **◀ ▶**：切换上/下一个液滴编号（1→8 循环）
- **下一个**：切换到下一个液滴并设置
- **回到1**：快速回到液滴 1

### 5.2 设置起点和终点

在 **起点/终点 模式**下：

1. **设置起点**：点击一个空闲格子，自动设为当前液滴的起点（蓝色，显示 S{编号}）
2. **设置终点**：再次点击另一个空闲格子，自动设为当前液滴的终点（橙色，显示 T{编号}）
3. **取消起点/终点**：点击已设置的起点或终点格子，即可取消该设置

> 智能点击逻辑：未设起点时点击 → 设为起点；已有起点时点击 → 设为终点；点击自己的起点/终点 → 取消。

### 5.3 支持多液滴

- 最多可配置 **8 个液滴**
- 每个液滴有独立编号（1-8）和路径颜色
- 切换编号后分别设置各自的起点和终点

### 5.4 液滴信息显示

```
┌───────────────────────────────────┐
│       起点: (1,2)  │  目标: (3,5)  │
└───────────────────────────────────┘
已配对: 2 / 8  已配置: 3 个液滴
```

- 卡片容器浅灰背景，圆角边框
- 起点信息蓝色，目标信息橙色
- 下方显示配对统计（起点+终点都设置才算配对）

### 5.5 清除操作

| 按钮 | 说明 |
|------|------|
| **清除当前** | 清除当前液滴的起点和终点，保留其他液滴 |
| **清除全部** | 清除所有液滴的起点和终点，保留障碍物 |

## 6. 障碍物设置

### 6.1 进入障碍物模式

点击「障碍物」按钮（选中变红色），进入障碍物设置模式。

### 6.2 标记障碍物

- **设置障碍物**：点击空闲格子 → 变为深色障碍物
- **取消障碍物**：再次点击已设置的障碍物 → 恢复空闲

### 6.3 清空障碍物

点击「清除障碍」按钮，移除所有障碍物格子。

> **提示**：障碍物模式与起点/终点模式互斥，切换模式会自动取消另一种模式的选择状态。

## 7. 路径规划与执行

### 7.1 路径规划

```
[规划路径]  [执行路径]  [停止]
```

点击 **「规划路径」**（蓝色）触发 **A\* 寻路算法**：

1. 自动收集所有已配对的液滴（起点+终点都已设置）
2. 使用 A\* 算法（曼哈顿距离启发式）计算四方向最短路径
3. **多液滴并行规划**：按路径长短排序（短路径优先），依次规划并标记已占用电极，避免路径冲突
4. 在网格上以不同颜色叠加显示每条路径
5. 右侧「路径信息」面板显示规划结果

#### 规划结果显示示例

```
规划结果: 2/3 成功
  液滴1: (0,0)→(2,3) (6步)
  液滴2: (1,1)→(5,5) (9步)
  液滴3: (0,7)→(4,6) 无路径
```

#### 失败诊断

当路径规划失败时，系统会自动诊断原因：

| 失败原因 | 说明 | 解决建议 |
|---------|------|---------|
| 起点被障碍物阻挡 | 起点位于障碍物上 | 更换起点位置 |
| 终点被障碍物阻挡 | 终点位于障碍物上 | 更换终点位置 |
| 起点等于终点 | 起点和终点在同一格 | 设置不同的起点和终点 |
| 路径被其他液滴占用 | 其他液滴路径占用了通道 | 调整起点/终点位置或顺序 |
| 无可达路径 | 障碍物完全包围了起点/终点 | 减少障碍物或调整布局 |

### 7.2 路径执行

#### 执行前准备

- 确保串口已连接
- 已规划好路径（直接点「执行路径」也会自动规划）
- 配置好执行间隔（每步延迟时间）

#### 执行间隔设置

```
执行间隔: [500    ▾] 毫秒/步
```

可选：100ms / 200ms / 500ms / 1000ms

- **100ms**：快速执行，适合调试
- **500ms**：标准速度（默认）
- **1000ms**：慢速，便于观察每一步

#### 执行过程

点击 **「执行路径」**（绿色）：

1. 自动执行 A\* 路径规划
2. 逐液滴依次运行：从液滴 1 开始，按路径顺序开关电极
3. **执行逻辑**：每一步打开当前电极 → 关闭前一步电极
4. 到达目标后自动切换到下一个液滴
5. 全部完成后状态变为「● 全部完成」

#### 执行过程观察

- 底部「运行日志」面板实时显示每一步操作
- 顶栏状态标签显示当前执行的液滴编号
- 右侧「路径信息」显示执行进度

### 7.3 停止

点击 **「停止」**（红色）：
- 立即停止定时器
- 发送 `ALLOFF` 关闭所有继电器
- 恢复界面交互（按钮、网格可用）

- 输入通道编号（0-47）

```
[输入指令...               ]  [发送]
```

- 在输入框中输入任意指令文本

## 11. 常见问题

### 11.1 串口相关问题

**Q: 扫描不到串口？**

A: 可能原因：
- 未安装 USB 转串口驱动（如 CH340/CP2102 驱动）
- 数据线仅供电无数据功能
- 以管理员身份运行软件

**Q: 串口连接失败？**

A: 检查：
1. 串口未被其他程序占用
2. 波特率设置与下位机一致（默认 115200）
3. 硬件连接正常
4. 点击「刷新」重新扫描

**Q: 发送指令后无响应？**

A: 检查：
- 串口状态是否为「已连接」
- 底部日志面板是否有收发记录
- 串口监视区是否显示返回数据

### 11.2 网格相关问题

**Q: 新建网格后之前的配置丢失？**

A: 新建网格会完全重建，所有起点/终点/障碍物/路径都会被清除。如需保留配置，请先保存工程文件。

**Q: 无法设置起点/终点？**

A: 确认以下状态：
- 当前处于 **起点/终点模式**（按钮应选中）
- 点击的是空闲格子（非障碍物、非其他液滴的起点/终点）

### 11.3 路径相关问题

**Q: 规划路径失败怎么办？**

A: 根据失败提示处理：
| 提示 | 处理方式 |
|------|---------|
| 障碍物阻挡 | 调整或减少障碍物 |
| 其他液滴占用 | 调整液滴起点/终点位置 |
| 起点等于终点 | 设置不同的起点和终点 |
| 无可达路径 | 检查是否被完全包围 |

**Q: 路径执行到一半想中断？**

A: 点击「停止」按钮，所有继电器会立即断开（发送 ALLOFF）。

**Q: 执行路径前必须点「规划路径」吗？**

A: 不需要。直接点「执行路径」会自动完成规划再执行。「规划路径」只预览不下发串口。

### 11.4 其他问题

**Q: 如何彻底重置所有配置？**

A: 点击「重置」按钮或菜单「文件 → 新建工程」。

**Q: 程序无法启动？**

A: 确保系统满足要求：Windows 10/11，安装包以管理员权限安装。

**Q: 路径颜色与液滴编号的对应关系？**

A: 路径颜色按液滴编号固定（共 8 种颜色）：蓝、朱红、翠绿、紫、橙黄、青、粉红、靛蓝。右侧面板图例区有显示。

## 附录

### A. 电极索引映射

电极索引计算公式：`index = row × 总列数 + col`

以默认 6×8 网格为例：

```
      Col 0  Col 1  Col 2  Col 3  Col 4  Col 5  Col 6  Col 7
Row 0:   0      1      2      3      4      5      6      7
Row 1:   8      9     10     11     12     13     14     15
Row 2:  16     17     18     19     20     21     22     23
Row 3:  24     25     26     27     28     29     30     31
Row 4:  32     33     34     35     36     37     38     39
Row 5:  40     41     42     43     44     45     46     47
```

### B. 工程文件结构

工程文件为 JSON 格式，示例结构：

```json
{
  "version": "2.2.3",
  "grid": { "rows": 6, "cols": 8 },
  "droplets": {
    "1": { "start": [0, 0], "target": [3, 5] },
    "2": { "start": [1, 2], "target": [4, 7] }
  },
  "obstacles": [[2, 3], [2, 4]],
  "current_droplet_id": 1
}
```

### C. 术语说明

| 术语 | 说明 |
|------|------|
| DMF | 数字微流控（Digital Microfluidics） |
| 液滴（Droplet） | 抽象概念代表一个从起点到终点移动的物体 |
